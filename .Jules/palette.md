## 2025-02-23 - Focus States and ARIA Labels on Icon Buttons
**Learning:** Found several icon-only buttons (both custom `<Button>` components and native `<button>` elements) lacking descriptive ARIA labels, making them inaccessible to screen readers. Additionally, the primary button component and native buttons used for table actions lacked explicit focus states (`focus-visible`), impairing keyboard navigation visibility.
**Action:** Always verify that icon-only interactive elements contain localized `aria-label`s. Ensure that all interactive elements, whether abstracted UI components like `<Button>` or raw HTML like `<button>`, apply standard focus state utility classes (`focus-visible:outline-2 focus-visible:outline-offset-2 focus-visible:outline-[var(--accent)]`) for consistent keyboard navigation.
